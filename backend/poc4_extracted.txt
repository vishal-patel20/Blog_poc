2.  Part A — Security Audit  
You will be given access to a deliberately vulnerable PHP application (DVPA — Damn Vulnerable PHP App). Your task is to 
identify, exploit (in your local environment only), document, and fix all vulnerabilities. This simulates a real -world 
penetration test and remediation cycle.  
 
IMPORTANT: All exploitation activity must be performed ONLY against the provided local application. Never 
test security vulnerabilities against live systems, third -party websites, or any system you do not own. Doing so 
is illegal.

3.2  Session Security  
Control  Implementation Required  
Session regeneration  Call session_regenerate_id(true) immediately after successful login  
Session fixation prevention  Destroy old session, generate new ID — never accept session ID from URL  
Absolute timeout  Store login timestamp in session — force re -login after 8 hours regardless of activity  
Idle timeout  Track last_activity in session — force re -login after 30 minutes of inactivity  
Secure cookie flags  Set session.cookie_secure=1, session.cookie_httponly=1, 
session.cookie_samesite=Strict  
Session data validation  Verify user agent and IP match session on every request (with documented trade -
off) 
 
3.3  CSRF Protection  
CsrfToken.php  
final class CsrfToken  
{ 
    public static function generate(): string  
    { 
        if (empty($_SESSION['csrf_token'])) {  
            $_SESSION['csrf_token'] = bin2hex(random_bytes(32));  
        } 
        return $_SESSION['csrf_token'];  
    } 
 
    public static function validate(string $token): bool  
    { 
        // hash_equals prevents timing attacks  
        return isset($_SESSION['csrf_token'])  
            && hash_equals($_SESSION['csrf_token'], $token);  
    } 
}

3.4  Additional Security Controls to Implement  
Control  Requirement  
Brute force protection  Lock account for 15 min after 5 failed login attempts — store attempt count in DB  
Input validation  Whitelist -validate all inputs — never use blacklist approach

4.  Security Headers Implementation  
SecurityHeaders.php  
// SecurityHeaders.php — apply on every response  
 
final class SecurityHeaders  
{ 
    public static function apply(): void  
    { 
        // Prevent clickjacking  
        header('X -Frame-Options: DENY');  
 
        // Prevent MIME type sniffing  
        header('X -Content-Type-Options: nosniff');  
 
        // Control referrer information  
        header('Referrer -Policy: strict -origin-when-cross-origin');  
 
        // Prevent XSS via strict CSP  
        header("Content -Security -Policy: default -src 'self'; " .  
               "script-src 'self'; style -src 'self' 'unsafe -inline'; " .  
               "img-src 'self' data:; font -src 'self'; " .  
               "connect -src 'self'; frame -ancestors 'none'");  
 
        // Remove PHP version disclosure  
        header_remove('X -Powered-By');  
    }
