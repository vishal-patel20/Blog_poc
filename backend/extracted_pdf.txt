3.  Functional Requirements  
Your API must expose the following endpoints. All responses must be JSON. All errors must return appropriate HTTP status 
codes.  
 
Method  Endpoint  Description  
GET /api/posts  Return paginated list of all blog posts  
GET /api/posts/{id}  Return a single post by ID — 404 if not found  
POST  /api/posts  Create a new post — validate all required fields  
PUT /api/posts/{id}  Full update of an existing post

4.1  Required OOP Patterns  
Pattern  Where to Apply It  
Singleton  Database connection class — only one PDO instance allowed  
Repository Pattern  PostRepository.php — separates data access from business logic  
Interface  Declare a RepositoryInterface — PostRepository implements it  
Abstract Class  BaseModel — shared properties and methods for all models  
Trait Timestampable trait — adds created_at / updated_at to any model  
Exception Handling  Custom exceptions caught in index.php, returned as JSON errors

Confidential — Internal Use Only  Page 4 5.  Git & GitHub Workflow Requirements  
You must demonstrate professional version control practice throughout this POC. Your commit history, branch structure, 
and pull requests are part of the assessment.  
 
REQUIREMENT: All code must live in a GitHub repository. You must share the repository link in your 
submission. A single commit with all code will not be accepted.  
 
Required Branch Strategy  
Branch  Purpose  
main  Production -ready code only — never commit directly to main  
develop  Integration branch — merge feature branches here first  
feature/router  Build the custom HTTP router  
feature/post -crud Implement Post model, controller, repository  
feature/comments  Implement Comment endpoints  
feature/validation  Add input validation and custom exceptions  
 
Commit Message Convention  
Conventional Commits  
# Format: <type>(<scope>): <short description>  
 
feat(router):    add support for PATCH method routing  
fix(post):       return 404 when post is soft -deleted  
refactor(db):    extract query builder into BaseRepository  
test(post):      add unit tests for PostRepository::findById  
docs(readme):    document API endpoints with curl examples  
chore(composer): add PHPUnit dependency  
 
Pull Request Requirements  
• Every feature branch must be merged via a Pull Request — not a direct push  
• Each PR must have a description explaining what was built and why  
• At least 2 PRs must include a self -review comment on your own code  
• Resolve at least one intentional merge conflict between two branches
